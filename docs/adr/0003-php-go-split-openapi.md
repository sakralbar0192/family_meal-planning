# ADR 0003: Распределение сервисов по PHP и Go, контракты OpenAPI

## Статус

Принято

## Контекст

Команда использует **PHP** (Symfony или Laravel) и хочет освоить **Go** на границах с внешним миром и горячих путях. Нужен единый способ описывать и эволюционировать HTTP API между сервисами и BFF.

## Решение

### 1. Распределение по языкам

| Сервис | Язык | Обоснование |
|--------|------|-------------|
| `recipe-import` | **Go** | Исходящий HTTP, жёсткие deadlines (NFR 3–5 с), отмена контекста, простой деплой бинаря. |
| `meal-planning` | **Go** (рекомендация) | Частые обновления слотов (DnD), предсказуемая латентность. Допустимо реализовать на PHP при желании единого стека CRUD — тогда обновить этот ADR. |
| `identity-access` | **PHP** | Регистрация, пароли, сессии, привычный стек. |
| `recipe-catalog` | **PHP** | CRUD, связи рецепт–ингредиенты, ORM. |
| `shopping-list` | **PHP** | Агрегация и оркестрация вызовов; при узком месте ядро можно вынести в Go позже. |
| `bff-web` | **PHP или Go** | Выбор реализации команды; один язык на компонент достаточно. |

Любой сервис может быть переписан при сохранении **OpenAPI** и поведения тестов.

### 2. OpenAPI как источник правды контракта

- Каждый микросервис публикует спецификацию **OpenAPI 3.x** в репозитории (каталог `contracts/openapi/`).
- Версия API в пути: `/v1/...`.
- Изменения, ломающие клиентов, оформляются как новая версия (`/v2`) или согласованный deprecation-цикл (записывается в ADR при первом breaking change).

### 3. Проверка контрактов

- CI: валидация YAML (spectral или openapi-cli), при наличии реализации — контрактные тесты (Prism mock или generated clients).
- BFF и микрофронты потребляют **объединённую** документацию BFF ([bff-routes.md](../../contracts/bff-routes.md) + при необходимости отдельный `bff.openapi.yaml`).

### 4. Импорт → сохранение (связь с контрактами)

- MVP: `recipe-import` возвращает **черновик** в теле ответа; постоянная запись только через `recipe-catalog` после явного «Сохранить» пользователем (см. [business-doc.md](../business-doc.md) сценарий 1).
- Опционально позже: оркестрация в BFF — отдельное решение и обновление OpenAPI.

## Последствия

- Два рантайма в CI (PHP и Go), два набора зависимостей.
- Плюс: явные контракты облегчают параллельную разработку фронта и бэка.
